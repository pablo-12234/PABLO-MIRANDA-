# PABLO-MIRANDA-PROYECTO
## DESCRIPCION PROYECTO
- Cliente: "Hola, necesito una aplicación web para gestionar los pedidos de mi tienda en línea. Quiero que los usuarios puedan registrarse, iniciar sesión y realizar pedidos de productos. También quiero que el administrador pueda gestionar los pedidos y los productos."
### -METODOLOGIA UTILIZADA
#### He elegido la metodología Scrum: 
porque es flexible y adaptable a cambios en el proyecto. Además, es ideal para equipos pequeños y medianos.
### -ETAPAS DEL PROYECTO: ROLES, DISTRIBUCION Y REUNIONES 
#### Sprint 0: 
Planificación Roles: Product Owner (PO), Scrum Master (SM), Equipo de Desarrollo (ED) Reuniones: Reunión de planificación del sprint Productos: Documento de requisitos, Plan de proyecto. 
#### Sprint 1:
Desarrollo Roles: ED Reuniones: Reunión diaria, Reunión de revisión del sprint Productos: Código fuente, Documentación del código. 
#### Sprint 2: 
Pruebas Roles: ED, QA (Aseguramiento de la calidad) Reuniones: Reunión de pruebas, Reunión de revisión del sprint Productos: Informe de pruebas, Código fuente actualizado. 
#### Sprint 3: 
Despliegue Roles: ED, Operaciones Reuniones: Reunión de despliegue, Reunión de revisión del sprint Productos: Aplicación desplegada, Documentación de despliegue.
### -REQUISITOS
#### Requisitos funcionales:
Registro de usuarios
Inicio de sesión
Realización de pedidos
Gestión de pedidos
Gestión de productos
#### Requisitos no funcionales:
Seguridad
Escalabilidad
Usabilidad
Herramientas necesarias:
IDE: Visual Studio Code
Lenguaje de programación: JavaScript (Node.js)
Herramientas de documentación: JSDoc
Herramienta de pruebas: Jest
Control de versiones: Git
### -CONCLUSION: 
#### El proyecto se llevará a cabo en 3 sprints. En el sprint 0, se realizará la planificación y se definirán los requisitos. En el sprint 1, se desarrollará la aplicación. En el sprint 2, se realizarán las pruebas y se corregirán errores. En el sprint 3, se desplegará la aplicación. Se utilizarán herramientas como Visual Studio Code, Node.js, JSDoc, Jest y Git para llevar a cabo el proyecto.
